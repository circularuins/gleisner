# Idea 008: Auto Translation — 多言語自動翻訳

**Status:** Raw idea
**Date:** 2026-03-17

## Summary

アーティスト/ファンが母国語でプロフィールや投稿を書きつつ、世界中のユーザーには自動翻訳で届ける仕組み。Gleisner の「消滅への抵抗」「探求」の哲学と合致する——言語の壁で作品が届かないのは理不尽な障壁。

## Motivation

- 日本語で書きたいが、世界にも発信したい（創業者自身のニーズ）
- 既存 SNS（X/Grok翻訳, Instagram, TikTok）は全て「翻訳ボタン」方式を採用済み
- プロフィール・投稿の両方に同じ仕組みが使える

## 想定 UX

### 「翻訳ボタン」方式（SNS 標準パターン）

1. 投稿/プロフィール保存時に、テキストの言語を自動検出（`Accept-Language` or NLP）
2. 閲覧者の端末 locale と異なる場合、「🌐 Translate」ボタンを表示
3. タップで翻訳テキストをインライン展開（原文も残す）
4. 再タップで原文に戻す

### 適用スコープ

| 対象 | 翻訳対象テキスト |
|------|-----------------|
| プロフィール | bio, displayName は対象外（固有名詞） |
| アーティストページ | bio, tagline |
| 投稿 | タイトル, 本文 |
| コメント | コメント本文 |

## コスト調査（2026年3月時点）

### API 料金比較

| サービス | 料金（per 1M chars） | 無料枠 | 備考 |
|----------|---------------------|--------|------|
| Google Cloud Translation v2/v3 | $20 | 500K chars/月 | 最も実績あり |
| Amazon Translate | $15 | 2M chars/月(12ヶ月) | 最安 |
| DeepL API Pro | $25 + $5.49/月 | 500K chars/月 | 高品質だが割高 |
| Google LLM Translation | $20 (in+out) | — | 高ニュアンス |

### コスト試算（ざっくり）

- プロフィール bio: 平均 200 文字
- 投稿本文: 平均 300 文字
- 1ユーザーが1日10投稿を閲覧 → 3,000 chars/day
- 1万 DAU → 30M chars/月 → Google で **$600/月**、Amazon で **$450/月**

### コスト最適化の方向性

1. **翻訳キャッシュ**: 同一テキスト → 同一翻訳。投稿は不変なので一度翻訳すれば再利用可能。プロフィールも更新頻度は低い。**キャッシュで実質コスト 1/10 以下にできる可能性大**
2. **オンデマンド翻訳**: 自動表示ではなくボタンタップ時のみ翻訳。全閲覧の翻訳率は 10-20% 程度と想定
3. **言語ペア限定**: 初期は日英のみ対応、需要に応じて拡大
4. **Edge/クライアントサイド翻訳**: ブラウザの Translation API（Chrome 限定）やオンデバイス翻訳を併用

キャッシュ + オンデマンドを組み合わせれば、1万 DAU でも **月 $50-100 程度** に抑えられる見込み。

## 先行事例

- **X (Twitter)**: 2025年に Google Translate から Grok 翻訳に移行。自社 AI で翻訳コストを内製化
- **Instagram / TikTok / Facebook**: 「翻訳を見る」ボタン方式。Meta は自社 NLLB モデルを使用
- 大手は全て自社モデルに移行する傾向 → 初期は API、スケール後に自社モデル検討

## 技術的な論点

- 言語検出の精度（短文は誤検出しやすい）
- 翻訳品質 vs コスト（Google/DeepL で十分か、LLM が必要か）
- キャッシュ戦略（テキストハッシュ → 翻訳結果の KV ストア）
- 翻訳結果の編集可否（アーティストが翻訳を手動修正できるか）
