# Idea 022: Pre-launch Checklist — Phased Launch Strategy

## Purpose

Gleisner のローンチを段階的に行うための、フェーズ別チェックリスト。

### Background

Gleisner は10歳の娘のアーティスト活動のライフログ・プロモーション用として構想が始まり、途中から SNS 構想に発展した。SNS としてのフル公開には法務・行政手続き・セキュリティ体制の整備が必要で、40〜200万円規模のコストが見込まれる。

**戦略**: SNS 機能はフルで実装を続けるが、公開は段階的に行う。

### Phase Definition

| Phase | 利用形態 | ユーザー登録 | 法的位置づけ |
|-------|---------|------------|------------|
| **Phase 0** | 家族数名のライフログ + 閲覧のみ一般公開 | 家族のみ（招待） | 個人サイト/ポートフォリオ |
| **Phase 1** | クローズドベータ（招待制で少数ユーザー追加） | 招待制 | グレーゾーン → 届出準備 |
| **Phase 2** | パブリック SNS | 一般公開 | 電気通信事業者 |

### Why Phase 0 Has Minimal Legal Requirements

Phase 0 では:
- **通信の媒介がない**: 投稿するのは家族のみ、一般閲覧者は読むだけ → 電気通信事業法の「他人の通信を媒介」に該当しない
- **不特定多数への提供ではない**: 家族内の利用は「他人の需要に応ずる事業」に該当しない
- **UGC プラットフォームではない**: 第三者のコンテンツを預からない → DMCA Safe Harbor、情プラ法、発信者情報開示の対象外
- **子どもの個人情報をプラットフォームが収集しない**: 親が自分の子のコンテンツを自分のサービスで公開 → COPPA は「プラットフォームが子どもからデータを収集する」場合の規制

法的には**個人のウェブサイト/ポートフォリオ**と同等。

## Phase 0: Family Lifelog + Public Viewing

**費用: 実質 0 円（既存のホスティング費用のみ）**

### Required (Phase 0)

- [ ] **最低限のサイト表記**: 運営者名、連絡先をフッター等に記載
- [ ] **外部送信の簡易開示**（電気通信事業法 27-12）
  - Cloudflare（CDN/R2）、Claude API（タイトル自動生成）の使用を明記
  - Google Analytics 等を使う場合はその旨も追記
  - 簡単な文章でサイト上に掲載すれば足りる（正式なポリシーページ不要）
- [ ] **HTTPS 有効確認**: Cloudflare Pages / Railway で TLS が有効であること
- [ ] **管理者認証の基本セキュリティ**: 家族アカウントのパスワード強度、JWT の基本的な保護

### Recommended (Phase 0)

- [ ] **ドメイン確保**: gleisner.app 等が取られていないか確認・確保
- [ ] **ソーシャルアカウント確保**: GitHub, X 等の名前を押さえる
- [ ] **DB バックアップ**: Railway の自動バックアップ設定確認（娘のライフログは取り返しがつかない）
- [ ] **EXIF メタデータ除去**: 写真アップロード時に GPS 座標等を自動除去（10歳の子どものプライバシー保護）

### Not Required in Phase 0

以下はすべて Phase 1+ に延期可能:

- ❌ 電気通信事業届出（通信の媒介がない）
- ❌ 利用規約（外部ユーザーがいない）
- ❌ 正式なプライバシーポリシー（個人情報を収集しない — 閲覧者のアカウント登録がない）
- ❌ 著作権・DMCA ポリシー（第三者の UGC がない）
- ❌ 児童安全ポリシー/COPPA 対応（プラットフォームが子どもの個人情報を収集していない）
- ❌ 発信者情報開示ポリシー（不特定多数の発信者がいない）
- ❌ AI 利用ポリシー（自分の子のコンテンツを自分が AI に送る — 第三者提供ではない）
- ❌ コミュニティガイドライン（コミュニティがない）
- ❌ 弁護士相談（法的リスクが極小）
- ❌ 商標登録（SNS として知名度が出る前は先取りリスク低い）
- ❌ サイバー保険（リスクが桁違いに低い）
- ❌ インシデント対応計画（個人サイトレベル）
- ❌ ペネトレーションテスト（攻撃面が極小）
- ❌ DMCA Designated Agent 登録（UGC プラットフォームではない）
- ❌ 事業体の決定（個人の活動）

## Phase 1: Closed Beta (Invite-only)

**費用目安: 5〜30万円**

招待制で家族以外のユーザーを受け入れ始める段階。ユーザー間の通信が発生し始めるため、法的義務が段階的に発生する。

### Trigger

家族以外の第三者にアカウントを発行し、投稿・DM・リアクション等のユーザー間機能を有効にした時点。

### Required (Phase 1)

**法務**:
- [ ] **弁護士の初回相談**（1〜3万円）: Phase 2 に向けた全体スコープの確認
  - 電気通信事業届出の要否確認（招待制・少数でも該当するか）
  - 事業体の相談（個人 vs 法人）
- [ ] **利用規約のドラフト**: 最低限の内容（自作 + 弁護士レビューは Phase 2）
- [ ] **プライバシーポリシーのドラフト**: ユーザーの個人情報を収集するため必須
- [ ] **AI 利用ポリシーの明記**: 他人のコンテンツを Claude API に送るため開示が必要

**セキュリティ（ADR 020 の Immediate + Short-term）**:
- [ ] JWT secret key logging 削除
- [ ] authMiddleware サイレント失敗修正
- [ ] パスワード長上限（128文字）
- [ ] セキュリティヘッダー（CSP, X-Content-Type-Options, X-Frame-Options 等）
- [ ] アカウント列挙防止（エラーメッセージ統一）
- [ ] GraphQL rate limiting（基本的なもの）
- [ ] CORS 本番設定（`*` 禁止）

**インフラ**:
- [ ] バックアップ復旧手順の文書化・テスト
- [ ] 依存パッケージ脆弱性スキャン有効化（Dependabot）

### Recommended (Phase 1)

- [ ] 商標登録出願の開始（審査 6〜7 ヶ月 → Phase 2 に間に合わせる）
- [ ] メール送信基盤の準備（SPF/DKIM/DMARC）
- [ ] ブルートフォース保護（ログイン試行制限）
- [ ] ファイルアップロード検証（マジックバイト、サイズ制限）

## Phase 2: Public SNS Launch

**費用目安: 40〜200万円（Phase 1 の費用を含む累計）**

一般ユーザーへの開放。電気通信事業者としての法的義務がフルに発生する。

### Trigger

招待制を解除し、不特定多数がアカウント登録できるようにした時点。

### Required (Phase 2) — Legal

**弁護士相談（27件の法的論点）**:

*著作権関連（ADR 018）*:
- [ ] LC-018-1: DMCA ポリシーページ・ToS 著作権条項のレビュー
- [ ] LC-018-2: IPFS テイクダウンの法的有効性（ADR 014 Phase 1 前）
- [ ] LC-018-3: DID + Ed25519 署名の法的証拠力（米/EU/日）
- [ ] LC-018-4: AGPL フォークの著作権侵害コンテンツ責任
- [ ] LC-018-5: 情報流通プラットフォーム対処法コンプライアンスチェックリスト
- [ ] LC-018-6: カウンター通知時の国境越え個人データ開示

*児童保護関連（ADR 019）*:
- [ ] LC-019-1: Email-plus VPC が COPPA 要件を満たすか
- [ ] LC-019-2: DID の「永続的識別子」扱い（COPPA 2025 改正）
- [ ] LC-019-3: 保護者同意記録アーキテクチャの法的妥当性
- [ ] LC-019-4: オーストラリア Social Media Minimum Age Act 免除可能性
- [ ] LC-019-5: 未成年者の音声・写真コンテンツの COPPA 対象範囲
- [ ] LC-019-6: 保護者虐待・親権紛争への法的義務
- [ ] LC-019-7: GDPR Art.8 年齢閾値の国別検出要否
- [ ] LC-019-8: 同意記録の保有期間

*電気通信事業法関連（ADR 022）*:
- [ ] LC-022-1: 届出書・ネットワーク構成図のレビュー
- [ ] LC-022-2: 通信の秘密 — スパム検出・コンテンツモデレーションの許容範囲
- [ ] LC-022-3: Claude API タイトル生成の同意要件
- [ ] LC-022-4: IPFS/リレーノード運用時の届出カテゴリ変更要否
- [ ] LC-022-5: 個人事業主での届出可否、法人格要否

*ポリシーアーキテクチャ関連（ADR 023）*:
- [ ] LC-023-1: Tier 1 全文書（D-1〜D-7）のドラフトレビュー
- [ ] LC-023-2: IP ログ保有期間 — 情プラ法最小要件 vs APPI データ最小化
- [ ] LC-023-3: 単一同意チェックボックスの APPI + GDPR 妥当性
- [ ] LC-023-4: Claude API データ送信の「第三者提供」該当性
- [ ] LC-023-5: 国際データ転送開示（Railway US + Cloudflare global）
- [ ] LC-023-6: 重大変更の定義（消費者契約法）
- [ ] LC-023-7: 保護者同意の分離クリックスルー要否（COPPA）

**行政手続き**:
- [ ] 電気通信事業届出（様式第8） — 届出書作成、ネットワーク構成図、提出、控え保管
- [ ] DMCA Designated Agent 登録（$6）
- [ ] 事業体の決定と手続き（個人事業主 or 法人設立）
- [ ] 暗号技術の輸出規制確認（Ed25519 の該非判定）

**ブランド保護**:
- [ ] 商標登録出願（Phase 1 で着手推奨 → Phase 2 までに登録完了が理想）

### Required (Phase 2) — Security

**ADR 020 の残り項目**:
- [ ] GraphQL API hardening（depth/complexity/rate limiting の本格版）
- [ ] Cloud KMS によるサーバーサイド鍵管理強化
- [ ] scrypt → Argon2id マイグレーション
- [ ] 2FA（TOTP + バックアップコード）
- [ ] GDPR minimum compliance（削除・エクスポートエンドポイント）
- [ ] デュアルトークン認証（access 15min + refresh 7日）— ADR 020 §2.6
- [ ] HTTPS 強制 + HSTS — ADR 020 §2.6.3
- [ ] セッション無効化 — ADR 020 §2.6.2

**追加セキュリティ項目（ADR 020 に未記載）**:
- [ ] ブルートフォース保護（認証エンドポイント専用）
- [ ] パスワードリセットトークンの有効期限・ワンタイム使用
- [ ] GraphQL BOLA（オブジェクトレベル認可）の全 resolver 検証
- [ ] ファイルアップロード検証（マジックバイト、サイズ制限、EXIF 除去）
- [ ] セキュリティイベントログ（ログイン成功/失敗、権限変更）
- [ ] breached password チェック（HaveIBeenPwned API）

### Required (Phase 2) — Operations

- [ ] **インシデント対応計画**: 対応手順書、漏洩通知フロー（APPI: 3-5日以内速報、GDPR: 72時間以内）、緊急連絡先リスト
- [ ] **コンテンツモデレーション体制**: コミュニティガイドライン策定、違反報告対応フロー（児童ポルノ、リベンジポルノ、DMCA）、発信者情報開示請求対応フロー
- [ ] **メール送信基盤**: 送信サービス選定、SPF/DKIM/DMARC 設定、特定電子メール法対応
- [ ] **保険**: サイバー保険・賠償責任保険の加入
- [ ] **ペネトレーションテスト**: OWASP Top 10 + GraphQL 固有の脆弱性チェック
- [ ] **バックアップ**: PostgreSQL + R2 のバックアップ戦略確定・復旧テスト

### Required (Phase 2) — UI

- [ ] `/legal` 配下の全ポリシーページ実装
- [ ] サインアップ時の同意チェックボックス
- [ ] 法的文書へのフッターリンク（全画面共通）
- [ ] ガーディアンフロー（ADR 019 実装時）

## Future: Post-Phase 2 Regulatory Triggers

Phase 2 以降、成長に応じて対応が必要になる項目:

| トリガー | 対応 |
|---------|------|
| **月間ユーザー 1,000 万超** | 電気通信事業の「大規模事業者」指定 → 登録制に変更 |
| **月間発信者 1,000 万超 or 投稿 200 万超** | 情プラ法の大規模プラットフォーム義務 |
| **課税売上 1,000 万超** | 消費税の課税事業者に |
| **投げ銭・チップ機能** | 資金移動業登録 or 前払式支払手段の検討 |
| **アプリ内通貨の未使用残高 1,000 万超** | 前払式支払手段の届出 + 供託義務 |
| **有料サブスクリプション** | 特定商取引法に基づく表記義務 |
| **EU 市場で 3 年超 or 売上 €10M 超** | EU Copyright Directive Art.17 小規模免除終了 |
| **オーストラリア進出** | Social Media Minimum Age Act 対応 |
